## 사용 방법

1. 처음 읽을 때: **01 → 07**로 기대치를 맞춘 뒤, **02 → 03 → 08**으로 데이터와 실제 호출 순서를 본다. LLM·RAG는 **04**, 튜닝은 **09**.
2. ActionLog·원정·저장 경로: **10**. 집무실·탐색 CLI: **11**. 파티·원정·캐릭터 생성: **12**. 관계/MCP 구현 진실: **13**.
3. 코드 수정 시: 동일 주제의 문서와 **08**(동작 변경 시 필수)을 스펙처럼 갱신한다.
4. 구현 위치: **[../GuildDialogue/](../GuildDialogue/)** — `Program.cs`, `Services/`, `Data/`, `Config/`.

## 구현 요약

- **대화 모드 1·2**: Ollama + 임베딩 인덱스 초기화 필수; 참조 지식은 **임베딩 RAG만** 사용한다.
- **원정·로그**: `DungeonRunSimulator`가 `ActionLog.json`에 쌓고, 옵션에 따라 `CharactersDatabase.json` 동기화.
- **캐릭터 생성(메뉴 5)**: 직업·스킬은 JSON 규칙, 배경·말투 등 서사는 Ollama.

루트 저장소의 Unity 본편과 연동할 때는 `ActionLog`·캐릭터 스키마를 맞추는 방향으로 확장하면 된다.
