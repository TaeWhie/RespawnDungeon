# 04. LLM 연동 설계

## 4.1 원칙

- **데이터 기반 (RAG 방식)**: 대화는 “검색된 데이터” 위에서만 한다. [02-데이터-설계](02-데이터-설계.md)의 캐릭터·던전 로그·메모리·현재 맥락을 조합한 컨텍스트만 넣고, **그 데이터 안에 있는 내용만** 말하도록 유도한다.
- **로직은 코드, 표현은 LLM**: 누가/언제/무슨 맥락인지는 코드가 정하고, 문장만 LLM이 생성.
- **하드 코딩 금지**: 키워드 목록·허용 주제 목록·N값·리트라이 횟수·상하한 등은 **설정 데이터**(SO, Config, JSON)에서 로드. 코드에 문자열/숫자 리터럴로 넣지 않는다.
- **로컬 우선**: Ollama 등 로컬 LLM 사용.
- **폴백 필수**: LLM 실패 시 규칙/템플릿으로 한 마디 생성.

## 4.2 RAG 흐름 요약

1. **Retrieve**: 화자 캐릭터, 최근 던전 로그, 메모리 요약, 직전 N턴, 최근 이벤트·현재 상황을 수집.
2. **Augment**: 위를 고정 슬롯(캐릭터/던전로그/메모리/최근턴/규칙)으로 문자열화해 프롬프트에 삽입.
3. **Generate**: “아래 컨텍스트만 보고 한 마디 생성. 컨텍스트에 없는 주제는 말하지 말 것.” 규칙을 명시하고 LLM 호출.

→ **데이터를 기반으로 대화하면 됨, RAG 마냥.**

### 컨텍스트 토큰 우선순위 (한계 시 생략 순서)

넣을 내용이 토큰 한도를 넘으면 **아래 순서로 뒤쪽부터 생략**한다. 구체적인 턴 수·회수는 **대화 설정**에서 로드(하드 코딩 금지).

1. **필수 유지**: 직전 발화(상대 한 마디), 현재 상황, 화자 이름·역할·말투 가이드
2. **우선 유지**: 직전 N턴 (설정 예: 3~5)
3. **축소 가능**: 던전 로그 회수·직전 턴 수를 설정에 정의된 축소값으로 변경
4. **가장 먼저 생략**: 장기 메모리 요약의 상세 문장

## 4.3 호출 시점·리트라이

- **한 턴 생성 시 1회** 호출. 아래 조건일 때만 **리트라이 최대 1~2회** 후 폴백.
  - **파싱 실패**: JSON 파싱 오류 또는 `line` 누락/빈 문자열
  - **반복 감지**: 직전 턴과 동일 문장 또는 동일 핵심어 2회 이상 반복. (핵심어 = `line`을 어절로 나눈 뒤 2글자 이상 명사/동사 등 규칙으로 정한 토큰. 동일어가 직전 2턴에 걸쳐 나오면 반복으로 간주.)
  - **주제 이탈 플래그**(선택): 아래 허용 주제 화이트리스트에 없는 내용이 `line`의 중심이 됨. (line의 중심 = 추출한 명사/핵심어 중 빈도 또는 위치 기준 상위 1~3개. 이들이 허용·컨텍스트와 한 개도 겹치지 않으면 이탈.)
- 입력: [02-데이터-설계](02-데이터-설계.md)의 “대화 요청 시 필요한 입력”을 **RAG 컨텍스트**로 조합한 프롬프트

**주제 이탈 검사 — 허용 키워드 (화이트리스트)**  
`line`이 허용 주제·컨텍스트와 무관한 내용만 담으면 이탈로 간주할 수 있다.

- **허용 주제 목록**: 코드에 하드 코딩하지 않는다. **설정 데이터**(SO, 설정 테이블, JSON)에서 로드. (문서 예시: 유물, 포션, 함정, 던전, 층/구간, 보스, 루팅, 파티, 전투, 인벤토리, 식당/아지트, 다음 출격, 부상/회복, 길드 마스터 지시, 동료와의 협력)
- **검사 방법**: `line`에서 명사/핵심어 추출(규칙은 설정에서 로드) 후, **설정에서 읽은** 허용 목록 또는 현재 컨텍스트(최근 이벤트·직전 발화)에 등장한 단어와 1개라도 겹치면 통과. 전부 미등록이고 맥락과 무관하면 이탈 플래그 → 리트라이 또는 폴백(리트라이 최대 횟수도 설정에서 로드)

## 4.4 프롬프트 구조 (안)

1. **시스템**
   - 역할: “중세 판타지 길드원의 한 마디 생성”
   - 출력 형식: JSON 등 고정 (예: `tone`, `intent`, `line`, `innerThought`)
   - 금지: 현대 용어·메타 발언·형식 깨기. **대사만 출력** (행동 묘사·괄호 설명 금지. `line`에는 말한 내용만).

2. **유저(또는 단일 프롬프트)**
   - [캐릭터] 이름, 역할, 배경, 말투 가이드([02](02-데이터-설계.md) 성격→말투 요약)
   - [상대](동료 대화 시) 상대 이름, 역할 (최소 정보)
   - [상태] 기분/신뢰/관계(선택), 직전 발화, 최근 이벤트, 현재 상황
   - [기억] 던전 로그 요약, 메모리 요약
   - [최근 턴] 직전 N턴 텍스트
   - [규칙] **컨텍스트에 있는 내용만 사용할 것.** 주제 유지, 상대 말에 반응, 말끝 스타일(예: -해/-해요 통일), 반복 금지. **필요하면 질문이나 제안을 한 마디에 넣어도 됨** (능동적 응답).
   - [예시] JSON 예시 1개 (선택, 로컬 LLM 품질 개선에 유리)

## 4.5 응답 형식

- **JSON 한 덩어리**: `{ "tone": "...", "intent": "...", "line": "...", "innerThought": "..." }`
- **line**: 실제 출력할 한 마디 (1~2문장)
- **tone/intent**: 로그·분석용 또는 다음 턴 힌트용

## 4.6 후처리

- **필수**: JSON 파싱, `line` 추출, 이스케이프/빈 문자열 처리
- **선택**: 말끝 통일(-해/-해요), 추상어 치환(이번 건 → 구체 키워드), 상대 이름 오기 제거, 반복 구간 제거

## 4.7 폴백

- LLM 미호출/실패/파싱 실패 시:
  - 직전 발화·이벤트 키워드로 짧은 문장 생성
  - **캐릭터별 기본 말투**(한두 가지 패턴)는 **캐릭터 데이터** 또는 **대화 설정**의 폴백 말투 목록에서 로드. 코드에 리터럴로 넣지 않는다.
  - 출력은 동일 구조(tone, intent, line)로 반환
