# 09. 설정 키 완전 참조

단일 소스: `GuildDialogue/Data/DialogueSettings.cs`, 기본값·예시: `Config/DialogueSettings.json`.  
JSON 키는 **PascalCase**(System.Text.Json 기본)로 직렬화된다.

---

## 9.1 루트

| 키 | 타입 | 설명 |
|----|------|------|
| `TraitToToneKeywords` | `Dictionary<string,string>` | 성격 축 이름(예: `Courage`) → 말투 힌트 한 줄. `PromptBuilder`가 사용. |
| `Ollama` | 객체 | 아래 9.2. |
| `Retrieval` | 객체 | 아래 9.3. 없으면 코드에서 `new RetrievalSettings()`로 채움. |
| `QualitativeAnalysis` | 객체 | 아래 9.4. |

---

## 9.2 `Ollama` (`OllamaSettings`)

| 키 | 기본(코드) | 설명 |
|----|------------|------|
| `BaseUrl` | `http://localhost:11434` | Ollama HTTP 베이스. |
| `Model` | `llama3.2` | `/api/generate` 모델 이름. |
| `EmbeddingModel` | null | 비우면 **`Model`을 임베딩에도 사용**. 채팅 전용 모델이면 임베딩 실패 가능 → **`nomic-embed-text` 등 별도 지정 권장**. |
| `TimeoutSeconds` | 60 | HTTP 타임아웃(초). |
| `Temperature` | 0.7 | 생성 온도. |
| `TopP` | 0.9 | nucleus sampling. |
| `NumPredict` | 0 | Ollama `num_predict`. 0이면 모델 기본. |
| `AnalysisStrategyMaxTokens` | 0 | 보조 분석 호출 등에서 쓰는 상한(0이면 해당 경로에서 미사용 또는 기본). |

## 9.3 `Retrieval` (`RetrievalSettings`)

### RAG 질의·검색

| 키 | 기본 | 설명 |
|----|------|------|
| `MaxEpisodicCharsInQuery` | 2800 | RAG 질의 문자열에 넣는 에피소드 버퍼 최대 글자. |
| `ExcludeCurrencyLineFromLoreEmbedding` | false | (로어 임베딩 질의에서) 통화/골드 라인 제외 여부. |
| `PrioritizeEpisodicMentionsInRag` | false | 에피소드에 언급된 엔티티 우선 처리 등(구현체 `GameKnowledgeRetriever` 참고). |
| `RagSearchPoolSize` | 12 | 코사인 상위 k개 청크. |
| `RagIncludeSpeakerLoadout` | true | 질의에 화자 인벤·장비·스킬 이름 혼합. |

### ActionLog → 프롬프트

| 키 | 기본 | 설명 |
|----|------|------|
| `LatestActionLogEntriesInPrompt` | 28 | **줄글 모드가 꺼졌을 때** 시스템 프롬프트에 넣는 JSON 꼬리 N건(Order 기준). |
| `UseActionLogNarrativeProse` | true | true면 원정 단위 **한국어 줄글** 블록. false면 위 꼬리 포맷. |
| `ActionLogNarrativeMaxDungeonRuns` | 8 | 줄글 모드에서 최근 던전 런 최대 개수(0이면 제한 없음). |
| `ActionLogNarrativeMaxChars` | 14000 | 줄글 블록 최대 글자(초과 시 앞부분 잘림). |

### 관점 기억

| 키 | 기본 | 설명 |
|----|------|------|
| `UsePerspectiveMemoryInPrompt` | true | 화자별 ActionLog 기반 주관 기억 블록. |
| `PerspectiveMemoryMaxLinesPerCharacter` | 24 | 화자당 최근 N줄(0이면 전체). |
| `PerspectiveMemoryExportPath` | null | 비어 있지 않으면 초기화 시 관점 줄을 JSON으로 덤프. **Config의 `ActionLog.json`과 같은 경로면 덤프 스킵**(형식 깨짐 방지). |

### 콘솔 세션 → ActionLog

| 키 | 기본 | 설명 |
|----|------|------|
| `PersistDialogueSessionsToActionLog` | true | 메뉴 1·2 종료 시 워킹 메모리 대화를 `ActionLog.json`에 Base 이벤트로 추가. |

### 길드 집무실 세만틱 가드레일

| 키 | 기본 | 설명 |
|----|------|------|
| `UseGuildOfficeSemanticGuardrail` | true | 임베딩 앵커로 메타/오프/원정 유사도 계산. 끄면 키워드 위주 폴백. |
| `GuildOfficeSemanticGuardrailThreshold` | 0.78 | 메타·오프월드 “히트”로 볼 최소 코사인. |
| `GuildOfficeSemanticDisambiguationMargin` | 0.06 | 메타/오프가 원정과 겹칠 때 원정 쪽으로 None 내리기 위한 마진. |
| `GuildOfficeSemanticExpeditionDisambigMin` | 0.58 | 원정 유사도가 이 값 이상일 때만 위 “경쟁” 로직 적용. |
| `GuildOfficeSemanticStrongAtypicalFloor` | 0.88 | 메타/오프 임베딩이 매우 높으면 원정과의 경쟁으로 None 내리지 않음. |
| `GuildOfficeSemanticMetaOffEmbeddingTieEpsilon` | 0.015 | 메타≈오프 코사인 차가 이보다 작으면 임베딩만으로는 구분 안 함 → None 후 키워드 타이브레이크. |
| `UseGuildOfficeObviousKeywordTieBreak` | true | 임베딩이 애매할 때 명백한 한글 단서로 메타/오프 승격. |
| `GuildOfficeExpeditionContextThreshold` | 0.72 | 길드장 발화가 “원정/작전 맥락”에 가까운지 볼 때 원정 앵커 임계값. |
| `GuildOfficeExpeditionUseRelativeEmbeddingGate` | true | true면 `exp >= 임계`뿐 아니라 `exp >= max(meta,off) + Lead`일 때만 deep(상대 게이트). |
| `GuildOfficeExpeditionDeepContextLead` | 0.03 | 상대 게이트 리드. |
| `UseGuildOfficeLlmIntentRouter` | false | 애매할 때 소형 LLM으로 의도 분류(토큰 증가). `--explore-guild-office` 러너는 이걸 강제로 끌 수 있음. |

## 9.5 튜닝 시 주의

- **`EmbeddingModel`**: 메뉴 1·2 초기화·RAG·가드레일이 **같은 임베딩 클라이언트**를 공유한다. 바꾸면 유사도 스케일이 달라져 `GuildOffice*` 임계값을 다시 맞출 수 있다.
- **`PerspectiveMemoryExportPath`**: `ActionLog.json`과 동일 파일 금지(코드가 거부).
