# 02. 데이터 설계

본 문서는 **`MiniProjects/GuildDialogue/Config`** 및 **`Data/*.cs`** 스키마를 기준으로 한다. 값의 일부는 `DialogueSettings.json`의 `Retrieval` 등으로 조절한다.

---

## 2.1 RAG·대화의 데이터 원칙

1. **Retrieve**: 워킹 메모리, 에피소드 버퍼, 화자 캐릭터, `ActionLog` 꼬리/줄글, (선택) 관점 메모리.
2. **Augment**: `PromptBuilder`가 블록 단위로 조합. 게임 DB 도감은 **임베딩 RAG**로 질의와 유사한 청크만 주입.
3. **Generate**: Ollama `/api/generate` — JSON 한 객체(`line` 등) 위주.

**하드 코딩 최소화**: 키워드·임계값·토글은 가능한 한 **`DialogueSettings.json`** 및 각종 JSON에 둔다. 코드는 로더·빌더가 읽어 반영한다.

---

## 2.2 캐릭터 (`CharactersDatabase.json` / `Character`)

| 필드 | 설명 |
|------|------|
| `Id` | 영문 식별자. `master`는 길드장 예약. |
| `Name` | 표시 이름. |
| `Age`, `Role` | 나이, 직업 식별자(`JobDatabase.RoleId`와 매칭). |
| `PartyId` | `PartyDatabase.json`의 `PartyId`와 연결. |
| `CurrentLocationId`, `CurrentLocationNote` | 아지트 시설 `BaseId` 등. |
| `Career` | 모험/관련 경력(년), 선택. |
| `Background`, `SpeechStyle` | 서사·말투 규칙. |
| `Mood` | 짧은 감정 라벨(런타임 갱신 가능). |
| `Likes`, `Dislikes` | 선택 목록. |
| `RecentMemorableEvent` | 최근 기억 한 줄(원정 후 갱신 등). |
| `Personality` | 10축 수치(`Courage`, `Caution`, … `Frugality`). |
| `Stats` | `CharacterStats` — HP/MP, 공방, 스탯, `StatPoints`. |
| `Inventory` | `ItemName`, `ItemType`, `Count`. |
| `Equipment` | 슬롯별 장비명. |
| `Relationships` | `TargetId`, `Affinity`, `Trust`, `RelationType`. |
| `Skills` | 스킬 이름 문자열 목록 — **`SkillDatabase.json`·`JobDatabase`와 정합**. |

**로드 시**: `DialogueConfigLoader.LoadCharactersWithJobSkillFilter()`가 `JobSkillRules`로 직업 밖 스킬을 제거한다.

---

## 2.3 직업·스킬 (`JobDatabase.json`, `SkillDatabase.json`)

- **`JobDatabase.json`**: `RoleId`, `DisplayName`, `Description`, `AllowedSkillNames[]`.
- **`SkillDatabase.json`**: 스킬명, 타입, 태그, 설명, 효과, MP, 쿨다운 등.
- **규칙**: `JobSkillRules` — `Character.Role`에 해당하는 `JobRoleData`가 있으면 `Skills`를 `AllowedSkillNames`에만 남긴다.

---

## 2.4 월드·참조 DB

| 파일 | 용도 |
|------|------|
| `WorldLore.json` | 세계·길드·던전 목록·조직 등. RAG 청크·프롬프트 서사에 사용. |
| `MonsterDatabase.json`, `TrapTypeDatabase.json`, `ItemDatabase.json` | 도감·RAG·시뮬. |
| `BaseDatabase.json` | 아지트 시설 `BaseId`, 이름, 설명. |
| `EventTypeDatabase.json` | 이벤트 코드 라벨. |
| `PartyDatabase.json` | 파티 ID, 멤버, 호칭. |
| `SemanticGuardrailAnchors.json` | 길드 집무실 메타/오프월드 판별용 앵커 문장(임베딩). |

`GameReferenceBundle`에 위 항목이 묶여 `PromptBuilder`·임베딩 인덱스 빌드에 쓰인다.

---

## 2.5 행동 로그 (`ActionLog.json` / `TestDataRoot`)

- **루트 타입**: `TestDataRoot` — `ActionLog: List<ActionLogEntry>` (이름은 역사적이며, 실질적으로 **타임라인 루트**).
- **`ActionLogEntry`**: `Order`, `Type`(`Dungeon`|`Base`), 파티, 던전명, 층, 전투·전리·함정·대화 등 구조화 필드. 줄글 `Summary`로 저장하지 않는 것이 원칙(시뮬·파서가 객체로 쌓음).
- **줄글 모드**: `Retrieval.UseActionLogNarrativeProse`가 true면 시스템 프롬프트에 **원정 단위 한국어 서사**로 넣을 수 있음(`ActionLogNarrativeFormatter`).

**필드·병합·세션 저장·트러블슈팅**은 [10-ActionLog-스키마-수명주기.md](10-ActionLog-스키마-수명주기.md)를 본다.

---

## 2.6 대화 설정 (`DialogueSettings.json`)

주요 그룹:

- **`Ollama`**: `BaseUrl`, `Model`, **`EmbeddingModel`**(예: `nomic-embed-text`), 타임아웃, `Temperature`, `TopP`, `NumPredict`.
- **`TraitToToneKeywords`**: 성격 축 → 말투 힌트 문구(프롬프트).
- **`Retrieval`**: RAG 질의 길이, `RagSearchPoolSize`, ActionLog 꼬리/줄글/최대 글자, 관점 메모리, 길드 집무실 세만틱 가드레일 임계값, 원정 딥 컨텍스트 게이트, **`UseAethelgardThreeLayerPrompt`**, **`UseMcpRuntimeToolFacts`** 등.
- **`QualitativeAnalysis`**: (선택) 프롬프트/응답 로깅.

자세한 필드 목록은 소스 `Data/DialogueSettings.cs`와 [06-구현-참조-GuildDialogue.md](06-구현-참조-GuildDialogue.md)를 본다.

---

## 2.7 캐릭터 생성(콘솔) 전용

- **`CharacterCreationRng`**: 나이(16~50), 경력(나이 상한·저분포 가중), 접수대 `BaseId`·메모 — **수치만** 담당.
- **`CharacterCreationLlmGenerator`**: Ollama에 JSON(`id`,`name`,`background`,`speechStyle`)만 받도록 프롬프트. `WorldLore`·접수 시설·기존 Id/이름 중복 금지.
