# 05. 품질 및 평가

## 5.1 목표 품질 (제품 관점)

| 영역 | 기대 |
|------|------|
| **사실성** | 인벤·파티·명단·도감과 모순되지 않음. RAG·MCP 블록을 “근거”로 인식하게 함. |
| **페르소나** | `SpeechStyle`, 성격, 직업 설명이 한 사람으로 유지. |
| **주제** | 길드·던전·아이템·동료 맥락 이탈 시 프롬프트의 제약·가드레일이 억제. |
| **기억** | `ActionLog` 줄글/에피소드·관점 메모리가 “최근 원정” 서사를 지원. |

## 5.2 정성 평가 참고

- 저장소 루트의 [RP 챗봇 상세 평가 가이드라인](../../RP%20챗봇%20상세%20평가%20가이드라인.md) — 말투, 지식 경계, 기억, 장면 전환 등(있을 경우).

## 5.3 개발·운영 체크리스트 (본 레포 기준)

**환경**

- [ ] Ollama 실행, `DialogueSettings.json`의 `Model`·`EmbeddingModel` 설치 여부.
- [ ] 메뉴 1·2 실행 전 임베딩 인덱스 빌드가 로그상 성공하는지.

**데이터**

- [ ] `CharactersDatabase.json`의 `Role`이 `JobDatabase.RoleId`와 맞을 때 스킬이 직업 정책과 일치하는지.
- [ ] `ActionLog`·캐릭터 동기화(`--gen-actionlog`, 원정 후 저장)가 의도대로인지.

**대화**

- [ ] JSON `line` 파싱 실패가 빈번하지 않은지(모델·프롬프트 길이 점검).
- [ ] 길드장 모드에서 집무실 가드레일·원정 딥 프롬프트 전환이 설정값과 어긋나지 않는지.

**캐릭터 생성**

- [ ] Ollama 실패 시 DB에 쓰이지 않는지.
- [ ] 생성된 배경·말투가 직업·스킬·나이와 모순 없는지(샘플 검수).

## 5.4 (선택) 정량·로그

- `QualitativeAnalysis` 설정이 켜져 있으면 프롬프트/응답 일부를 콘솔·파일에 남길 수 있음 — 부하·개인정보 유의.
