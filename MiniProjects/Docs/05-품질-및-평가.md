# 05. 품질 및 평가

## 5.1 목표 품질

- **자연스러움**: 사람끼리 대화하는 느낌, 자기 할말만 하지 않음
- **주제 유지**: 유물/포션/함정 등 현재 맥락을 놓치지 않음
- **캐릭터 일관성**: 말투·가치관이 성격 수치·배경과 맞음
- **기억 반영**: 배경·던전 로그를 적절히 언급

## 5.2 평가 참고

- [RP 챗봇 상세 평가 가이드라인](../../RP%20챗봇%20상세%20평가%20가이드라인.md)  
  - 말투 고유성, 지식 경계, 행동 양식, 기억력, 장면 전환, 능동적 대화, 지시문 준수 등

## 5.3 개발 시 체크리스트

**대화 품질**
- [ ] 직전 상대 발화에 직접 반응하는가?
- [ ] RecentEvent/Opening 주제(유물·포션 등)가 유지되는가?
- [ ] 같은 문장·같은 핵심어가 연속 반복되지 않는가?
- [ ] 캐릭터별 말투(제안형, 감탄, 손익 언급 등)가 구분되는가?
- [ ] 던전 로그·배경이 맥락에 맞게 쓰이는가?
- [ ] 필요 시 질문/제안을 넣는 능동적 한 마디가 나오는가? (RP 가이드라인: 능동적 대화)

**설계 보완 반영**
- [ ] **하드 코딩 금지**: 키워드 목록·허용 주제·N값·리트라이 횟수·오프셋 상하한 등은 모두 설정(SO/Config/JSON)에서 로드 — [02](02-데이터-설계.md) 2.0, [04](04-LLM-연동-설계.md) 4.1
- [ ] **대화 설정 스키마**: [02](02-데이터-설계.md) 2.6 필드(MaxRecentTurns, KeywordCategories, AllowedTopicKeywords, FallbackLines 등)를 설정에서 로드
- [ ] **폴백 말투**: 캐릭터 데이터 또는 대화 설정의 폴백 문장 목록에서 로드 — [04](04-LLM-연동-설계.md) 4.7
- [ ] N 값 적용: 직전 턴·던전 로그 회수 등은 **대화 설정**에서 읽기 (예: 8/5, 5/3) — [02](02-데이터-설계.md)
- [ ] 성격 수치 → 말투 가이드 문자열 변환 후 [캐릭터]에 포함 — [02](02-데이터-설계.md)
- [ ] 동료 대화 시 상대(B) 이름·역할 컨텍스트 포함 — [03](03-대화-플로우.md)
- [ ] 리트라이 조건: 파싱 실패/반복/주제 이탈 시 최대 1~2회 후 폴백 — [04](04-LLM-연동-설계.md)
- [ ] 턴 저장 시 장기 메모리 갱신(intent/line 기반) — [02](02-데이터-설계.md) 2.4
- [ ] 관계/신뢰/기분 오프셋: 키워드 목록 + ± 적용 및 상·하한 — [02](02-데이터-설계.md) 2.4
- [ ] 주제 이탈 검사: 허용 키워드 화이트리스트·핵심어 추출 규칙으로 `line` 검사 후 리트라이/폴백 — [04](04-LLM-연동-설계.md) 4.3
- [ ] 대화 재개 시 세션 턴·장기 요약·시나리오 키 로드로 컨텍스트 복원 — [03](03-대화-플로우.md) 3.5
- [ ] 토큰 한계 시 컨텍스트 우선순위대로 생략 — [04](04-LLM-연동-설계.md)
- [ ] LLM 실패 시에도 폴백으로 자연스러운 한 마디가 나오는가?
