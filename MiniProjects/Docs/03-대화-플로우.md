## 3.1 실행 진입점

### 대화형 메뉴 (`dotnet run` 인자 없음)

| 선택 | 모드 | Ollama |
|------|------|--------|
| **1** | 아지트 현황 — 베이스 활동 랜덤 배정(대화 1쌍 필수) 후 동료 **2인**이 **총 2~4발화**(번갈아) | 필요(초기화 시 임베딩 + 턴마다 생성) |
| **2** | 길드장 직접 참여 | 동일 |
| **3** | 파티 편성 | 불필요 |
| **4** | 원정(던전 시뮬) | 불필요 |
| **5** | 캐릭터 생성 | 서사 생성에 필요 |
| **0** | 종료 | — |

메뉴 **1·2**는 `DialogueManager.InitializeAsync()`를 탄다. 이 안에서 **임베딩 인덱스 빌드가 실패하면 예외**로 중단될 수 있다(Ollama + `EmbeddingModel` 필수).

### CLI 모드 (첫 인자로 분기)

| 인자 | 동작 |
|------|------|
| `--explore-guild-office` | `GuildOfficeExplorationRunner` — 집무실 시나리오 탐색(임베딩·가드레일 설정 따름). |

## 3.2 `DialogueManager` 한 세션(요약)

1. **초기화**  
   - 캐릭터 로드(`LoadCharactersWithJobSkillFilter`), 월드·아이템·`GameReferenceBundle`, `ActionLog` 타임라인.  
   - 에피소드 버퍼 빌드(`MemoryManager`).  
   - **임베딩 인덱스** 전면 구축(실패 시 예외).  
   - 옵션: 길드 집무실 세만틱 가드레일 웜업.

2. **턴 루프** (모드 1 또는 2)  
   - 화자·청자 결정(`PartyRosterResolver` 등).  
   - 워킹 메모리 + RAG 블록(`ResolveReferenceKnowledgeRagAsync` — 임베딩만).  
   - `PromptBuilder`로 시스템/유저 프롬프트 조합.  
   - Ollama 응답 파싱(JSON `line`).  
   - 세션 로그·선택적 ActionLog 저장, 사회적 인상·관계 델타 등.

3. **길드장 모드(2)**  
   - 에델가드 3층 구조, MCP 런타임 팩트(`AethelgardMcpRuntimeFacts`), 집무실 토픽 게이트·가드레일이 설정에 켜져 있으면 활성.

## 3.3 메뉴 1 — 아지트 허브(`BaseHubScreen`)

- `BaseDatabase.json`의 시설·`AvailableServices`로 활동 풀을 만들고, **아지트에 있는 동료**( `CurrentLocationId`가 베이스 `BaseId`이거나 비어 있음)마다 **무작위 활동**을 붙인다.
- **대화**는 항상 **정확히 한 쌍**(무작위 2명)에게만 배정되며, 표시 후 `DialogueManager.RunBasePairDialogueAsync`로 **총 2~4발화**(번갈아) 생성한다.
- 구 **관전 6인 고정**(카일·리나·브람) 흐름은 제거되었다.

---

## 3.4 파티·원정(메뉴 3·4)

- **파티**: `PartyDatabase.json` 읽기/쓰기, 멤버 `Character.PartyId` 정합.  
- **원정**: `DungeonRunSimulator`가 로그를 쌓고, 종료 후 `CharacterMoodUpdater` 등으로 기분·결과 반영 가능. `ActionLog` 병합·캐릭터 DB 저장은 `DialogueConfigLoader` 경로로 처리.

---

## 3.5 캐릭터 생성(메뉴 5)

1. 직업 선택(`JobDatabase`).  
2. 허용 스킬만 선택.  
3. RNG로 나이·경력·접수대 위치.  
4. **Ollama**로 Id·이름·배경·말투 JSON 수신 — 실패 시 저장 안 함.  
5. `JobSkillRules` 재적용 후 `CharactersDatabase.json`에 append.

---

## 3.6 세션 복구·저장

- 콘솔 세션은 프로세스 메모리 위주; `PersistDialogueSessionsToActionLog`가 켜져 있으면 종료 시 대화를 `ActionLog`에 Base 타입 등으로 남길 수 있음.  
- 장기 “세이브 파일” 형태의 전용 슬롯은 Unity 연동 시 확장 여지.
