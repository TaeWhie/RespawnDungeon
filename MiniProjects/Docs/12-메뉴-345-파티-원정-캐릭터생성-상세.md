# 12. 메뉴 3·4·5 상세 — 파티 · 원정 · 캐릭터 생성

Ollama 없이 동작하는 경로(3·4)와 Ollama 필수 경로(5)를 분리해 기술한다. 구현: `PartyManagementConsole.cs`, `DungeonRunSimulator.cs`, `CharacterCreationConsole.cs`, `CharacterCreationLlmGenerator.cs`.

---

## 12.1 메뉴 3 — 파티 편성 (`RunPartyRosterMenuAsync`)

- **대화형 하위 메뉴**: 목록 / 새로 만들기 / 수정 / 삭제 / 상위로.
- **PartyId 규칙**: 영문·숫자·`_`·`-`만. 빈 입력 시 `party_` + 8 hex 자동 제안.
- **멤버 선택**: 번호 공백 구분 또는 `all`. 목록에는 `master` 제외.
- **저장 파일**: `PartyDatabase.json`.
- **캐릭터 동기화**: `ApplyPartyMembership`이 `Character.PartyId`와 각 `PartyData.MemberIds`를 **양방향 정리**한다. 한 캐릭터는 한 파티만 — 다른 파티에 넣으면 이전 소속에서 제거·`MemberIds` 정리.
- **삭제**: 파티 레코드 삭제 + 해당 `PartyId`를 쓰던 캐릭터의 소속 해제.

---

## 12.2 메뉴 4 — 원정 (`RunExpeditionAsync`)

1. 등록된 파티 중 선택(`SelectParty`).
2. **시드**: 정수 또는 엔터(무작위) — `DungeonRunSimulator.Generate(inputs, seed)`.
3. **캐릭터 DB 동기화 여부**: Y/n — `SaveCharactersDatabase(sim.CharactersAfterRun)`.
4. **ActionLog 처리**:  
   - 엔터(기본): **기존 `ActionLog.json` 뒤에 병합**(`SaveActionLogAfterSimulation(..., replaceExisting: false)`).  
   - `n` 입력: **이번 원정만으로 덮어쓰기**(`replaceExisting: true`와 동일 계열).
5. **후처리**: `CharacterMoodUpdater.ApplyAfterDungeonRun` — 기분·결과 반영.

CLI `--gen-actionlog`는 동일 시뮬을 쓰되 인자로 출력 경로·시드·`--party`·`--replace-actionlog`·`--no-sync-chars`를 받는다(`Program.cs`).

---

## 12.3 메뉴 5 — 캐릭터 생성 (`RunCreateCharacterAsync`)

1. `JobDatabase.json` 비어 있으면 종료.
2. 직업 번호 선택 → 설명 출력.
3. **스킬**: 직업 `AllowedSkillNames` 안에서만 — 쉼표 번호, `all`, `none`.
4. **RNG**: 나이(`CharacterCreationRng.RollAge`), 경력(`RollCareerYears`), 접수 시설 `BaseId`·메모(`BaseDatabase.json` 기준).
5. **Ollama**: `CharacterCreationLlmGenerator.TryGenerateAsync` — JSON `id`, `name`, `background`, `speechStyle`(camelCase) 등. 실패 시 **저장하지 않음**.
6. **후처리**: `JobSkillRules`로 스킬 재검증, 로스터에 append 후 `SaveCharactersDatabase` 또는 `Characters.json` 경로(`DialogueConfigLoader` 규칙과 동일).

**의존성**: `DialogueSettings.json`의 `Ollama.BaseUrl` / `Model` — 임베딩은 캐릭터 생성 경로에서 필수 아님(생성기는 채팅 API만 사용).

---

## 12.4 메뉴 3·4·5와 대화 1·2의 연결

- 원정으로 쌓인 `ActionLog`·갱신된 인벤/스탯은 메뉴 1·2 초기화 시 `LoadTimelineData` / 캐릭터 로드로 **다음 대화에 반영**된다.
- 파티·`PartyId`는 `PartyRosterResolver`·던전 시뮬 입력에 사용된다 — 데이터만 맞추면 코드 변경 없이 시나리오 확장 가능(반면 **메뉴 1 출연진은 코드 고정** — [08](08-런타임-시퀀스-상세.md)).
