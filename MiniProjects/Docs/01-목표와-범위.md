# 01. 목표와 범위

## 1.1 프로젝트 목표

| 층위 | 목표 |
|------|------|
| **게임(본편)** | 길드가 던전을 탐험·의뢰를 수행하는 관리형 로그라이트. 플레이어는 길드 운영(탐험 AI·파티 등)에 집중한다. |
| **대화(미니프로젝트)** | 길드 마스터(GM)와 길드원, 길드원 상호 간 **데이터 기반** 대화. LLM은 **프롬프트에 실린 사실(캐릭터·로그·RAG·규칙)** 안에서만 말하도록 설계한다. |
| **운영 원칙** | 로컬 **Ollama**(채팅 + 임베딩)을 전제로 한다. 대화·RAG·길드 집무실 가드레일은 임베딩 모델이 없으면 초기화 단계에서 실패할 수 있다. |

## 1.2 구현 범위: `MiniProjects/GuildDialogue`

| 포함 | 설명 |
|------|------|
| **콘솔 앱** | .NET 6, `Config/**/*.json`을 소스 오브 트루스로 로드·저장. |
| **아지트 허브** | 메뉴 1 — 베이스 활동 랜덤 배정(대화 1쌍 필수) 후 동료 2인이 총 2~4발화(번갈아, Ollama). |
| **길드장 1:1** | 메뉴 2 — 사용자 입력과 NPC 1:1, 에델가드 3층 프롬프트·MCP 스타일 런타임 팩트·세만틱 가드레일 옵션. |
| **파티·원정** | 메뉴 3·4 — 파티 DB 편성, 던전 시뮬로 `ActionLog`·캐릭터 스탯/인벤 반영(Ollama 불필요). |
| **캐릭터 생성** | 메뉴 5 — 직업(`JobDatabase`)·스킬 선택 후 **Ollama로만** 서사(Id·이름·배경·말투) 생성. |
| **CLI 도구** | `--explore-guild-office`, `--explore-guild-office-llm`, `--gen-actionlog` 등. |

| 비포함(본편/향후) | 설명 |
|-------------------|------|
| Unity 런타임 내 직접 대화 UI | `Character` 등 DTO는 Unity `CharacterData`와 매핑 가능하도록 설계만 맞춤. |
| 상용 클라우드 LLM API | 기본 경로는 Ollama 로컬. |
| 키워드 전용 RAG 폴백 | 참조 지식은 **임베딩 검색만** 사용(구현 상 키워드 RAG 분기 제거). |

## 1.3 성공 기준 (대화 품질)

- **일관성**: 인벤·파티·명단과 모순되지 않도록 프롬프트에 표·검증 힌트·RAG를 넣는다.
- **페르소나**: `SpeechStyle`, 성격 수치, `JobDatabase` 직업 설명이 한 인물로 읽힌다.
- **기억**: `ActionLog`(에피소드·줄글 모드)·관점 메모리·RAG 도감이 맥락을 보강한다.
- **운영**: Ollama(채팅·`EmbeddingModel`)가 켜진 환경에서 대화 모드 1·2가 기동 가능하다.
