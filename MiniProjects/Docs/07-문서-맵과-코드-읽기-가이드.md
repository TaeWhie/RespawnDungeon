# 07. 문서 맵과 코드 읽기 가이드

**질문: `01`~`06` 문서만으로 이 미니 프로젝트 전체를 알 수 있는가?**

**답: 아니다.** 문서는 **의도·데이터 구조·주요 흐름**을 담고 있고, 아래 항목은 **소스가 단일 진실**이다. 이 파일은 **무엇이 문서에 있고, 무엇을 코드에서 봐야 하는지**를 구분한다.

| 내용 | 이유 | 읽을 파일(우선순위) |

| **메뉴 1 아지트 허브** | 베이스 활동 풀·2인 대화 쌍 | `BaseHubScreen`, `DialogueManager.RunBasePairDialogueAsync` |

## 7.4 “문서만 수정하고 싶다”면

- 동작 변경이 **코드에만 있고 문서에 없으면** 버그가 아니라 **문서 부채**다.
- 역할: **01~06·08~11**은 제품 설명, **코드 주석·`DialogueSettings.cs`**는 구현 세부.

이 레포의 정책: **중요한 행동 변화**(예: 관전 모드에 캐릭터 추가)는 **코드와 문서(08) 둘 다** 갱신하는 것이 안전하다.
