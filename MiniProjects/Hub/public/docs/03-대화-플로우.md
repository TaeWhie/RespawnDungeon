# 03. 대화 플로우

## 3.1 대화 모드

| 모드 | 화자 | 상대 | 용도 |
|------|------|------|------|
| 길드 마스터 대화 | 길드원 1명 | 길드 마스터(사용자) | 보고, 지시, 질문·답변 |
| 동료 대화 | 길드원 A ↔ B | 서로 | 아지트에서 둘이 대화 |

## 3.2 한 턴의 흐름

1. **입력 수집**
   - 누가 말하는가 (화자)
   - 직전에 누가 뭐라고 했는가 (상대 + 직전 발화 텍스트)
   - 최근 이벤트, 현재 상황
   - (선택) 오프닝 질문/한 마디 — 동료 대화 시 시작 문장

2. **컨텍스트 구성 (RAG Retrieve + Augment)**
   - **검색**: 화자 캐릭터(이름, 역할, 배경, 성격 수치), 던전 로그 요약, 장기 메모리 요약, 직전 N턴
   - **조합**: 위 데이터를 프롬프트용 텍스트 블록으로 합침. 이 블록이 “대화에 쓸 수 있는 전부”임을 명시
   - → LLM은 이 데이터 안에 있는 내용만 참고해 말함

3. **한 마디 생성**
   - 규칙/폴백만 쓸지, LLM을 부를지 결정
   - LLM 사용 시: 프롬프트 조합 → 호출 → 응답 파싱 → 후처리
   - 품질 검사(선택): 반복/주제 이탈/말투 이상 시 리트라이 또는 폴백

4. **출력 및 메모리 반영**
   - 생성된 한 마디를 턴으로 저장
   - 관계/신뢰/기분 오프셋, 미해결 안건 등 필요 시 갱신

## 3.3 동료 대화 특칙

- **턴 번갈아가며**: A가 말함 → B가 대답 → A가 대답 → …
- **직전 발화**: 항상 “상대의 마지막 한 마디”가 Incoming으로 들어감
- **상대 캐릭터 컨텍스트**: A가 말할 때는 **상대 B의 최소 정보**를 프롬프트에 포함한다. (이름, 역할, 필요 시 성격 말투 1문장). B의 던전 로그/장기 메모리는 생략 가능. → “지금 누구에게 말하는지”를 LLM이 알 수 있게 함.
- **메모리**: 동료 대화 세션용 턴만 따로 유지 (길드 마스터 대화 턴과 분리 가능)
- **말투**: 서로 반말/동료 말투. “~해 보자”, “~하면 좋겠어” 등 제안형 위주

## 3.4 시나리오 전환

- **이벤트/상황이 바뀌면** (예: “유물 분배” → “포션 사용 시점”)
  - 시나리오 키 갱신
  - 세션 턴 또는 최근 턴/안건 초기화
  - 다음 대화는 새 시나리오 기준으로 시작

## 3.5 대화 재개 (세션 복구)

- **중단 후 재개 시** 로드할 상태: 저장된 **세션 턴**(최근 N턴), **장기 요약**, **시나리오 키**. 이 세 가지로 컨텍스트를 복원한 뒤 다음 턴부터 이어서 생성.
- 동료 대화·GM 대화는 세션을 분리해 저장할 수 있으므로, 재개 시 “어떤 대화(세션 ID 또는 모드)”를 이어갈지 지정해 해당 턴·메모리만 로드.
